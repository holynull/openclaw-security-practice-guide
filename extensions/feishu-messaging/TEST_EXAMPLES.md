# Feishu Reminder 富文本上下文测试用例

## 🆕 测试场景 0：回复消息提醒（最简单！）

### 步骤：

1. **先发送一条普通消息：**

   ```
   我们需要优化部署流程：
   1. 使用滚动更新
   2. 增加健康检查
   3. 配置自动回滚
   ```

2. **直接回复这条消息（点击回复按钮）：**
   ```
   2分钟后提醒我这个方案
   ```

### 期望结果：

Agent 会**自动提取**被回复的消息内容：

```typescript
send_feishu_reminder({
  targetId: "ou_xxx",
  message: "⏰ 提醒：部署优化方案",
  details:
    '引用的消息：\n"我们需要优化部署流程：\n1. 使用滚动更新\n2. 增加健康检查\n3. 配置自动回滚"',
});
```

**重点：** 无需说"刚才说的"，只要你**回复消息**，Agent 就会自动把原消息内容放入提醒的 details 中！

## 测试场景 1：引用对话提醒

### 步骤：

1. **先发送一条消息建立上下文：**

   ```
   我们需要优化部署流程，主要有三点：
   1. 使用滚动更新而不是停机部署
   2. 增加健康检查机制
   3. 配置自动回滚
   ```

2. **然后发送提醒请求：**
   ```
   2分钟后提醒我刚才说的部署优化方案
   ```

### 期望结果：

Agent 应该调用：

```typescript
send_feishu_reminder({
  targetId: "ou_xxx",
  message: "⏰ 提醒：部署优化方案",
  details:
    "之前讨论的部署优化方案：\n1. 使用滚动更新而不是停机部署\n2. 增加健康检查机制\n3. 配置自动回滚",
});
```

发送的消息会是**富文本卡片**格式，包含：

- 蓝色标题："⏰ 提醒"
- 主要内容："部署优化方案"
- 上下文区域：完整的三点方案

## 注意事项

### 当前实现的触发条件：

Agent 会在以下情况使用 `details` 参数：

1. **🆕 用户回复消息（最可靠！）**
   - 系统检测到 `has_reply_context: true`
   - 自动提取被回复的消息内容到 `details`

2. **用户明确引用**
   - "刚才说的..."
   - "之前提到的..."
   - "那个问题..."
   - "那件事..."

3. **需要总结内容**
   - "今天讨论的任务"
   - "会议的议题"
   - "方案的要点"

4. **问题回复提醒**
   - "提醒我回复XX的问题"
   - "提醒我处理XX的事情"

### 如果没有触发富文本：

可能的原因：

1. Agent 没有识别到需要引用上下文
2. Skill 配置还需要更多示例引导
3. 对话距离太远，Agent 没有找到相关内容

### 调试方法：

检查 session 日志中的 `send_feishu_reminder` 调用：

```bash
ssh openclaw "tail -100 ~/.openclaw/agents/main/sessions/*.jsonl | grep -A 5 'send_feishu_reminder'"
```

查找 `details` 字段是否存在。

## 实际测试命令

如果想通过 CLI 直接测试（包含上下文）：

```bash
ssh openclaw "cd ~/github.com/holynull/openclaw && docker compose run --rm openclaw-cli agent --local --agent main --message '设置一个测试提醒：1分钟后提醒我关于API优化的三个要点：1.添加缓存 2.减少查询 3.使用CDN'"
```

但是 **更推荐通过飞书消息测试**，因为：

1. CLI 是单轮对话，没有历史上下文
2. 飞书消息有完整的对话历史
3. Agent 更容易从对话中提取相关内容
